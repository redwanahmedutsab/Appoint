# 🗓️ Appointly — Online Appointment Booking System

A production-ready appointment booking platform for Dhaka, Bangladesh.
Built with **Next.js 14** (frontend), **Laravel 11** (backend), and **PostgreSQL 16** (database).

---

## 🏗️ Project Structure

```
appointly/
├── frontend/          # Next.js 14 + TypeScript + Tailwind CSS
├── backend/           # Laravel 11 + PHP 8.2 REST API
├── nginx/             # Nginx reverse proxy config
└── docker-compose.yml # Full stack orchestration
```

## ⚡ Quick Start (Docker)

### 1. Clone and configure environment

```bash
git clone https://github.com/your-org/appointly.git
cd appointly

# Copy and edit backend env
cp backend/.env.example backend/.env
# Set APP_KEY, DB_PASSWORD, FRONTEND_URL, etc.

# Copy and edit frontend env
cp frontend/.env.example frontend/.env.local
# Set NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1
```

### 2. Generate Laravel app key

```bash
docker run --rm -v $(pwd)/backend:/app composer:latest \
  sh -c "cd /app && composer install && php artisan key:generate"
```

### 3. Start all services

```bash
docker-compose up -d --build
```

### 4. Run migrations and seed

```bash
docker exec -it appointly_backend php artisan migrate --seed
```

### 5. Access

| Service   | URL                          |
|-----------|------------------------------|
| Frontend  | http://localhost:3000        |
| API       | http://localhost:8000/api/v1 |
| Admin     | http://localhost:3000/admin  |

**Default admin credentials:**
- Email: `admin@appointly.com.bd`
- Password: `Admin@1234`

> ⚠️ Change the admin password immediately after first login.

## 🔧 Manual Setup (Without Docker)

### Backend (Laravel)

```bash
cd backend
composer install
cp .env.example .env
php artisan key:generate

# Configure your .env with PostgreSQL credentials
php artisan migrate --seed
php artisan storage:link
php artisan serve --port=8000
```

### Frontend (Next.js)

```bash
cd frontend
npm install
cp .env.example .env.local
# Set NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1
npm run dev
```

## 🗄️ Database

PostgreSQL 16 with the following core tables:

| Table                | Description                              |
|---------------------|------------------------------------------|
| `users`              | All users (customers, providers, admins) |
| `service_categories` | 5 predefined service categories          |
| `neighborhoods`      | ~67 Dhaka DNCC/DSCC neighbourhoods       |
| `provider_profiles`  | Business profiles with approval workflow |
| `services`           | Services offered by each provider        |
| `availability_slots` | Time slots generated by providers        |
| `bookings`           | Appointment bookings                     |
| `reviews`            | Post-booking ratings and reviews         |
| `commission_records` | 5% commission tracking                   |
| `favorite_providers` | User-saved providers                     |
| `disputes`           | Complaint management                     |

## 🔐 API Reference

Base URL: `http://localhost:8000/api/v1`

### Authentication

| Method | Endpoint          | Auth | Description       |
|--------|-------------------|------|-------------------|
| POST   | `/register`       | ❌   | User registration |
| POST   | `/login`          | ❌   | Login             |
| POST   | `/logout`         | ✅   | Logout            |
| GET    | `/me`             | ✅   | Get current user  |
| PATCH  | `/me`             | ✅   | Update profile    |

### Providers (Public)

| Method | Endpoint                        | Description               |
|--------|---------------------------------|---------------------------|
| GET    | `/providers`                    | List providers (filtered) |
| GET    | `/providers/{slug}`             | Provider detail           |
| GET    | `/providers/{id}/slots?date=X`  | Available slots           |
| GET    | `/providers/{id}/reviews`       | Provider reviews          |

### Bookings (Authenticated)

| Method | Endpoint                          | Description         |
|--------|-----------------------------------|---------------------|
| GET    | `/bookings`                       | User's bookings     |
| POST   | `/bookings`                       | Create booking      |
| GET    | `/bookings/{id}`                  | Booking detail      |
| POST   | `/bookings/{id}/cancel`           | Cancel booking      |
| POST   | `/bookings/{id}/reschedule`       | Reschedule booking  |
| POST   | `/bookings/{id}/review`           | Submit review       |

### Provider Dashboard

| Method | Endpoint                            | Description            |
|--------|-------------------------------------|------------------------|
| GET    | `/provider/dashboard`               | Stats overview         |
| POST   | `/provider/profile`                 | Create profile         |
| PATCH  | `/provider/profile`                 | Update profile         |
| GET    | `/provider/services`                | List own services      |
| POST   | `/provider/services`                | Create service         |
| POST   | `/provider/slots/generate`          | Generate time slots    |
| PATCH  | `/provider/bookings/{id}/status`    | Update booking status  |

### Admin Panel

| Method | Endpoint                            | Description               |
|--------|-------------------------------------|---------------------------|
| GET    | `/admin/dashboard`                  | Platform stats            |
| POST   | `/admin/providers/{id}/approve`     | Approve/reject provider   |
| GET    | `/admin/commissions`                | Commission records        |
| POST   | `/admin/commissions/settle`         | Mark commissions settled  |
| GET    | `/admin/commissions/export`         | Export CSV                |
| GET    | `/admin/disputes`                   | Dispute list              |
| PATCH  | `/admin/disputes/{id}/resolve`      | Resolve dispute           |

| Role       | Capabilities                                                              |
|------------|---------------------------------------------------------------------------|
| **User**   | Search providers, book appointments, review, manage favourites            |
| **Provider** | Manage services, availability, confirm/complete bookings, view earnings |

---

## 💰 Revenue Model

- **Commission**: 5% per completed booking (hardcoded per SRS)
- **Settlement**: Manual weekly via bank/bKash transfer
- **Tracking**: Admin exports CSV reports for each provider
- **No payment gateway** in Phase 1 (per scope)

## 🚀 Production Deployment

### Environment variables to set in production

```env
# Backend
APP_ENV=production
APP_DEBUG=false
APP_KEY=base64:your-generated-key
DB_PASSWORD=strong-password-here
SANCTUM_STATEFUL_DOMAINS=appointly.com.bd
FRONTEND_URL=https://appointly.com.bd

# Frontend
NEXT_PUBLIC_API_URL=https://appointly.com.bd/api/v1
```

### Deploy with Docker Compose

```bash
docker-compose -f docker-compose.yml up -d --build
docker exec appointly_backend php artisan migrate --force
docker exec appointly_backend php artisan config:cache
docker exec appointly_backend php artisan route:cache
```

## 🛡️ Security Features

- **Bcrypt** password hashing (ROUNDS=12)
- **Laravel Sanctum** token-based API auth
- **Role-Based Access Control** (RBAC) via custom middleware
- **Session expiry** after 30 minutes of inactivity
- **Server-side validation** on all inputs
- **Soft deletes** to preserve booking history
- **Race condition prevention** via DB-level locking on slot booking

- ❌ SMS/Email notifications
